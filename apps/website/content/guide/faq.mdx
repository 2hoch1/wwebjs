## Installation & Setup

<Accordions type="single">
  <Accordion title="Can I use multiple accounts at once?">
    Yes, run multiple app instances with different auth strategies (RemoteAuth with database works
    best for this).
  </Accordion>
  <Accordion title="What's the difference between headless and GUI mode?">
    - **Headless:** No visible browser, lower resource usage, limited video support - **GUI:** Shows
    browser window, useful for debugging, better media support
  </Accordion>
  <Accordion title="Can I run this on a VPS without display?">
    Yes, use headless mode. For video support, use `xvfb-run` on Linux to provide a virtual display.
  </Accordion>
</Accordions>

## Authentication

<Accordions type="single">
  <Accordion title="Do I need to scan QR code every time?">
    No, use LocalAuth (local file storage) or RemoteAuth (database) for session persistence.
  </Accordion>
  <Accordion title="What's the difference between NoAuth, LocalAuth, and RemoteAuth?">
    - **NoAuth:** Session lost on restart (testing only) - **LocalAuth:** Session saved to local
    `.wwebjs_auth` folder (single instance) - **RemoteAuth:** Session in cloud database (multiple
    instances)
  </Accordion>
  <Accordion title="My session keeps expiring">
    Ensure you're using an appropriate auth strategy with proper backup. Use RemoteAuth for
    stability.
  </Accordion>
</Accordions>

## Messages & Operations

<Accordions type="single">
  <Accordion title="How do I filter messages by sender?">
    Use `msg.from` property:

    ```javascript
    if (msg.from === '1234567890@c.us') {
      // Handle message
    }
    ```

  </Accordion>
  <Accordion title="Can I send formatted text (bold, italic)?">
    Yes:

    ```javascript
    await msg.reply('*bold* _italic_ ~strikethrough~')
    ```

  </Accordion>
  <Accordion title="How do I add a caption to media?">
    Pass caption as second argument:

    ```javascript
    const media = MessageMedia.fromFilePath('./image.jpg')
    await msg.reply(media, 'This is my caption')
    ```

  </Accordion>
  <Accordion title="How do I download media from messages?">
    Use `downloadMedia()`:

    ```javascript
    if (msg.hasMedia) {
      const media = await msg.downloadMedia()
      fs.writeFileSync('file.jpg', media.data, 'base64')
    }
    ```

  </Accordion>
  <Accordion title="Can I edit or delete messages?">
    Yes, but only recent messages (< 2 days):

    ```javascript
    await msg.edit('Edited text')
    await msg.delete(true) // Delete for everyone
    ```

  </Accordion>
</Accordions>

## Groups

<Accordions type="single">
  <Accordion title="How do I create a group?">
    Use `client.createGroup()`:

    ```javascript
    const group = await client.createGroup('Group Name', ['1234567890@c.us'])
    ```

  </Accordion>
  <Accordion title="Can I remove someone from a group?">
    Yes, if you're admin:

    ```javascript
    await group.removeParticipants(['1234567890@c.us'])
    ```

  </Accordion>
  <Accordion title="How do I check if I'm an admin?">
    Check participant roles:

    ```javascript
    const isAdmin = group.participants.find(p => p.id === myId)?.isAdmin
    ```

  </Accordion>
  <Accordion title="Can I restrict messages to admins only?">
    Yes:

    ```javascript
    await group.setMessagesAdminsOnly(true)
    ```

  </Accordion>
</Accordions>

## Performance & Scaling

<Accordions type="single">
  <Accordion title="My app is slow with many messages">
    Implement message queuing and database indexing for better performance.
  </Accordion>
  <Accordion title="How many apps can I run on one server?">
    Depends on server resources. Each app needs ~200MB+ RAM for browser.
  </Accordion>
  <Accordion title="How do I handle rate limiting?">
    Add delays between operations:

    ```javascript
    await msg.reply('Message 1')
    await new Promise(r => setTimeout(r, 1000))
    await msg.reply('Message 2')
    ```

  </Accordion>
</Accordions>

## Errors & Debugging

<Accordions type="single">
  <Accordion title="Cannot find module 'whatsapp-web.js'">
    Run `npm install whatsapp-web.js`
  </Accordion>
  <Accordion title="Connection refused errors">
    WhatsApp disconnected you. This is normal. Implement reconnection logic.
  </Accordion>
  <Accordion title="How do I debug my app?">
    Set `headless: false` to see browser, add console.log statements, use VS Code debugger.
  </Accordion>
  <Accordion title="Session invalid on restart">
    Use proper auth strategy (LocalAuth or RemoteAuth) instead of NoAuth.
  </Accordion>
</Accordions>

## Best Practices

<Accordions type="single">
  <Accordion title="Should I use await?">
    Yes, always. whatsapp-web.js is heavily asynchronous:

    ```javascript
    const contact = await msg.getContact() // Correct
    const contact = msg.getContact() // Wrong
    ```

  </Accordion>
  <Accordion title="What's the recommended way to structure my app?">
    Split into modules:

    <Files>
      <Folder name="app" defaultOpen>
        <Folder name="handlers" defaultOpen>
          <File name="message-handler.js" />
          <File name="command-handler.js" />
          <File name="event-handler.js" />
        </Folder>
        <Folder name="models">
          <File name="user.js" />
          <File name="conversation.js" />
        </Folder>
        <Folder name="utils">
          <File name="logger.js" />
          <File name="config.js" />
        </Folder>
        <File name="index.js" />
      </Folder>
    </Files>

  </Accordion>
  <Accordion title="Should I store sessions in the repository?">
    No, use `.gitignore`:

    ```
    # .gitignore
    .wwebjs_auth/
    .env
    node_modules/
    ```

  </Accordion>
</Accordions>
